# AGENTS.md — php-manager

Этот документ содержит контекст, полезный при работе с проектом AI-агентам.

## Что делает проект

`php-manager` — небольшая CLI-утилита на Go для генерации PHP/Laravel файлов. Запускается внутри PHP-проекта, находит `composer.json`, определяет PSR-4 namespace и через интерактивную TUI-форму создаёт нужный файл с правильным namespace и базовым шаблоном.

## Ключевые файлы

| Файл        | Назначение                                      |
|-------------|-------------------------------------------------|
| `main.go`         | Точка входа, форма выбора типа/final/папки/имени и запуск генерации |
| `suggest.go`      | Логика автодополнения пути к папке                    |
| `suggest_test.go` | Юнит-тесты автодополнения                             |
| `generate_test.go`| Юнит-тесты генерации PHP-файлов                       |
| `go.mod`          | Модуль Go и зависимости                                |
| `go.sum`          | Контрольные суммы зависимостей                         |
| `.gitignore`      | Исключения для Git                                     |

## Архитектура

- `main()` — точка входа: парсинг `composer.json`, бесконечный цикл формы выбора типа/папки/имени и генерации. Выход — по `Esc` / `Ctrl+C`.
- `findComposerRoot(start)` — подъём вверх от текущей директории до `composer.json`.
- `resolveNamespace(root, filePath)` — вычисление namespace по пути нового файла.
- `generate(kind, dir, name, final)` — генерация `.php` файла с шаблоном, `declare(strict_types=1);` и опциональным `final`.
- `suggest.go` — автодополнение пути к папке:
  - `suggestDirs(root, value)` возвращает подпапки, соответствующие текущему вводу.
  - `splitPath(value)` разделяет путь на базовую директорию и префикс.
- В форме добавлен `huh.NewConfirm` для выбора модификатора `final`.
- Кастомный `KeyMap` в `main.go` переназначает `Tab` на принятие подсказки, а `Enter` — на переход к следующему полю.
- Все генерируемые файлы содержат `declare(strict_types=1);`.

## Зависимости

Прямые:

- `github.com/charmbracelet/huh` — формы и ввод.
- `github.com/charmbracelet/lipgloss` — цветной вывод.

Перед изменением зависимостей запускать:

```bash
go mod tidy
```

## Сборка и запуск

```bash
# Сборка
go build -o php-manager .

# Запуск из папки PHP-проекта
./php-manager
```

## Стиль и соглашения

- Код и комментарии на русском языке (пользовательский интерфейс тоже).
- Минимальные изменения: проект небольшой, весь код в одном файле.
- Перед коммитом убедиться, что `go build` и `go mod tidy` отрабатывают без ошибок.

## Известные ограничения и TODO

- Код разбит по файлам, но всё ещё в пакете `main`; при росте функциональности стоит выделить пакеты (`generator`, `composer`, `ui`).
- Нет CI/CD.
- Нет CLI-флагов / неинтерактивного режима.
- Обработка ошибок в некоторых местах минимальна (`_` вместо проверки).
- Шаблоны файлов зашиты в код; нет пользовательских шаблонов.
- Нет проверки на перезапись существующего файла.
- Нет поддержки `autoload-dev` и других автозагрузок, кроме `psr-4`.
- Автодополнение папок работает только по префиксу текущего уровня; стрелки переключают подсказки, Tab принимает.
